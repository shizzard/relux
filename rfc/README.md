# Relux RFCs

Substantial changes to Relux are proposed and discussed through RFCs (Requests for Comments).

## Process

1. Draft an RFC as `RXXX-short-name.md` in this directory.
2. Open a pull request for discussion.
3. Update the status as the RFC progresses.

## Statuses

| Status        | Meaning                                              |
|---------------|------------------------------------------------------|
| `draft`       | Under development, not yet ready for review          |
| `active`      | Open for discussion and feedback                     |
| `accepted`    | Approved, ready for implementation                   |
| `implemented` | Fully implemented in the codebase                    |
| `rejected`    | Considered and declined                              |
