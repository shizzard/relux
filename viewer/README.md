# relux-viewer

Svelte 5 SPA that renders a Relux test report (`events.json` produced by `relux-runtime`).

The runtime inlines the gzipped bundle into per-test `event.html` files alongside the schema-typed `window.RELUX_DATA` payload, so reports open directly via `file://` with no server. The full surface (app bar, timeline, events list, detail panel with source/shell/variables/call-stack views, env/shells/artifacts modals, keyboard shortcuts) is documented in [`docs/reference/05-test-log-viewer.md`](../docs/reference/05-test-log-viewer.md).

## Layout

- `src/App.svelte`, `src/main.ts` — entry point. Mounts `App` against `window.RELUX_DATA` and inlines `styles/tokens.css` + `styles/global.css`.
- `src/components/` — UI components. Top-level rows and modals; `src/components/sections/` holds the detail-panel views (source, shell buffer, variables, call stack).
- `src/lib/` — non-UI logic: `state.svelte.ts` (`ViewerState`, the central reactive store), derivations (`derive.ts`, `flatten.ts`, `scope.ts`, `timeline.ts`), formatters (`format.ts`, `bif.ts`, `artifacts.ts`), source highlighting, clipboard/actions, theme.
- `src/types/` — TypeScript bindings generated from the Rust event schema by `ts-rs` (gitignored, regenerated by `just viewer-build`).
- `scripts/gzip-bundle.mjs` — post-build step that gzips `dist/relux-viewer.js` into `dist/relux-viewer.js.gz`.

## Build

The vendored, gzipped IIFE bundle lives at `vendor/relux-viewer.js.gz` (committed at the repo root). It is embedded into the `relux-runtime` binary via `include_bytes!`, so a fresh `cargo build` does not require any JS toolchain.

To regenerate the vendored bundle after editing anything under `viewer/`:

```sh
just viewer-build
```

That recipe (a) regenerates the TS schema from the Rust types via `ts-rs` (under `src/types/`), (b) runs `npm ci && npm run build` inside the `node:lts-slim` Docker image, and (c) copies the gzipped bundle to `vendor/relux-viewer.js.gz`.

A pre-commit hook (installed once via `just install-hooks`) and a CI step both verify that the vendored bundle stays in sync with `viewer/` sources.

## Tests and checks

```sh
just viewer-test     # vitest unit tests (derive, flatten, scope, timeline, artifacts, ...)
just viewer-check    # svelte-check (types + Svelte diagnostics)
```

Both run inside the `node:lts-slim` Docker image, mirroring CI.

## TypeScript schema

Types under `src/types/` are generated from the Rust event types by `ts-rs`, driven by the `ts-export` cargo feature on `relux-runtime`. They are regenerated as the first step of `just viewer-build` and gitignored. To regenerate them without rebuilding the bundle:

```sh
cargo test -p relux-runtime --features ts-export 'export_bindings_'
```
